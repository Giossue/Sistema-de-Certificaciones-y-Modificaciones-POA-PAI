# Criterios de aceptación globales

## Certificación POA/PAI

Una certificación es aceptable si:

- Está asociada a una unidad requirente.
- Está asociada a una actividad vigente del POA.
- Está validada contra una versión de cédula MEF.
- Tiene documentos habilitantes completos.
- Tiene saldo suficiente.
- No duplica una certificación vigente sobre la misma actividad.
- Tiene número automático.
- Tiene PDF generado.
- Tiene auditoría completa.

## Modificación POA

Una modificación POA es aceptable si:

- Registra actividad afectada.
- Registra campos modificados.
- Registra motivo.
- No modifica fuente.
- Valida nueva estructura contra cédula MEF.
- Genera informe técnico.
- Crea nueva versión POA al aprobarse.
- Reasocia certificaciones vigentes si corresponde.
- Registra auditoría completa.

## Saldos

El cálculo de saldo es aceptable si:

- Usa decimal exacto.
- Considera certificaciones vigentes.
- Considera liquidaciones modo A.
- Considera liquidaciones modo B como monto retirado para reforma.
- Considera anulaciones.
- Se expone desde un único módulo.
- Tiene pruebas unitarias para escenarios críticos.

## Auditoría

Una operación es aceptable si registra:

- Usuario.
- Rol.
- Entidad afectada.
- ID de entidad.
- Acción.
- Estado anterior.
- Estado nuevo.
- Fecha y hora.
- Motivo cuando aplique.
- IP o metadata técnica cuando esté disponible.
